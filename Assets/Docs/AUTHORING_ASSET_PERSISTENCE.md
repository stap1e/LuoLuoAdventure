# Authoring Asset Persistence

Date: 2026-06-27  
Unity: 2022.3.62f3 LTS

## Why this exists

The playable vertical slice depends on authored Unity `ScriptableObject` assets. These files may be generated by setup menu commands, but they are not disposable build artifacts: their `.asset` and `.meta` files carry stable GUIDs and tuned gameplay data that must survive a fresh clone.

Setup tools remain available and idempotent, but they are a recovery path, not the only source of truth.

## Assets that must be committed

### Mission definitions

Commit these files and their `.meta` files:

- `Assets/Data/Missions/ConvoyEnergyConflict.asset`
- `Assets/Data/Missions/BorderRetaliation.asset`
- `Assets/Data/Missions/CityGateDispute.asset`

These are authored `MissionDefinitionSO` assets used by validator coverage, demo flow documentation, and scene setup.

### AI behavior profiles

Commit both authoring assets and runtime Resources copies, plus all `.meta` files:

- `Assets/Data/AIProfiles/AggressiveRaider.asset`
- `Assets/Data/AIProfiles/DefensiveGuard.asset`
- `Assets/Data/AIProfiles/Negotiator.asset`
- `Assets/Data/AIProfiles/Hardliner.asset`
- `Assets/Data/AIProfiles/CommanderUnit.asset`
- `Assets/Data/AIProfiles/NeutralCivilian.asset`
- `Assets/Resources/AIProfiles/AggressiveRaider.asset`
- `Assets/Resources/AIProfiles/DefensiveGuard.asset`
- `Assets/Resources/AIProfiles/Negotiator.asset`
- `Assets/Resources/AIProfiles/Hardliner.asset`
- `Assets/Resources/AIProfiles/CommanderUnit.asset`
- `Assets/Resources/AIProfiles/NeutralCivilian.asset`

`AIBehaviorProfileSO.LoadDefault()` uses `Resources.Load("AIProfiles/{type}")`, so the Resources copies are runtime-loadable authored data.

### Combat tuning

Commit these files and their `.meta` files:

- `Assets/Data/Combat/CombatTuningConfig.asset`
- `Assets/Resources/CombatTuningConfig.asset`
- `Assets/Resources/SubFactionDatabase.asset`
- `Assets/Resources/AudioFeedbackProfile.asset`
- `Assets/Resources/WorldMarkerProfile.asset`

`CombatTuningConfigSO.LoadOrDefault()` uses `Resources.Load("CombatTuningConfig")`, so the Resources copy is runtime-loadable authored data. `GameBootstrap`, `AudioFeedbackService`, and `WorldMarkerService` also load the listed Resources assets by name.

## Assets/Data vs Assets/Resources

- `Assets/Data` is the authoring location for project data assets.
- `Assets/Resources` is only for assets that runtime code loads with `Resources.Load`.
- Mission definitions are currently not runtime-loaded through Resources, so they only need `Assets/Data/Missions` persistence.
- AI profiles and combat tuning currently have Resources copies because runtime code resolves them through `Resources.Load` fallbacks.

## Ignore policy

`Library/`, `Temp/`, `Logs/`, `TestResults/`, generated project files, and broad generated output remain ignored.

`.gitignore` now allowlists the required authored `.asset` and `.meta` files under:

- `Assets/Data/Missions`
- `Assets/Data/AIProfiles`
- `Assets/Data/Combat/CombatTuningConfig.asset`
- `Assets/Resources/AIProfiles`
- `Assets/Resources/CombatTuningConfig.asset`
- `Assets/Resources/SubFactionDatabase.asset`
- `Assets/Resources/AudioFeedbackProfile.asset`
- `Assets/Resources/WorldMarkerProfile.asset`

This keeps transient generated content out of git while allowing required gameplay data to be versioned.

## Regeneration commands

Use Unity menu items:

- `LuoLuoTrip/Setup/Create AI Behavior Profiles`
- `LuoLuoTrip/Setup/Create Mission Prototype Data`
- `LuoLuoTrip/Setup/Create Combat Tuning Config`
- `LuoLuoTrip/Setup/Create Commander Mission Prototype Scene`

These commands are intended to be idempotent:

- Missing assets are created.
- Existing tuned assets are preserved.
- Empty identity/readability fields may be backfilled.
- Existing numeric/boolean tuning is not reset.

## Audit commands

From the repository root:

```bash
git check-ignore -v Assets/Data/AIProfiles/AggressiveRaider.asset
git check-ignore -v Assets/Data/Missions/CityGateDispute.asset
git check-ignore -v Assets/Data/Combat/CombatTuningConfig.asset
git check-ignore -v Assets/Resources/AIProfiles/AggressiveRaider.asset
git check-ignore -v Assets/Resources/CombatTuningConfig.asset

git status --ignored --short Assets/Data Assets/Resources
git ls-files Assets/Data/Missions/CityGateDispute.asset
```

For required assets, `git check-ignore -v` should produce no output.

## Validator coverage

`VerticalSliceValidator.CheckAuthoringAssets` verifies:

- required asset existence;
- `.meta` existence;
- AI profile validation;
- mission `missionId`, `displayName`, objectives, and outcomes;
- combat tuning validation;
- git ignore status when git is available.

If git is unavailable, the validator reports a warning instead of failing core asset validation.

## Known limitations

- Data and Resources copies can drift if edited independently. This is intentional for now: setup commands preserve user tuning rather than forcibly syncing one copy over the other.
- Resources copies are only required where runtime code uses `Resources.Load`.
- Setup menu commands are recovery/idempotency tools; committed assets remain the expected source for reproducible fresh clones.
